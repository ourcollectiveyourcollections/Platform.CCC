# Legal and Compliance Guardrails

## Operating language

Use conservative language.

Prefer:

- profile record
- contribution record
- licence record
- royalty accounting record
- mint-ready metadata
- project library
- collection registry
- member profile
- asset provenance
- review required

Avoid until legal review:

- guaranteed return
- dividend entitlement
- investment product
- asset-backed yield
- passive income token
- profit share guarantee
- financial product claim

## Token stance

The early platform should not present tokens as guaranteed financial returns.

Start with:

```text
records
metadata
permissions
contribution logs
licence terms
mint-ready exports
```

Only move to live minting after review.

## Contract stance

The templates in this repo are scaffolds, not legal advice.

Each live use should be reviewed by a qualified lawyer if it involves:

- commercial licensing
- revenue distribution
- royalties
- government contracts
- public sales
- token issuance
- cross-border participants
- custody or wallet operations
